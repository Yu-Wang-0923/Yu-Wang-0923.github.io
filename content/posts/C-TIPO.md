## 什么是 C-TIPO？

C-TIPO 是一种结构化的提示词模板，全称是 **Context → Task → Input → Process → Output**。它借鉴了计算机科学的输入-处理-输出模型，专为 AI Agent 设计，能够清晰地传达任务的完整上下文和执行路径。

## 模板结构

```markdown
## Context（背景）
描述当前状态、为什么需要做这件事、相关的约束条件。

## Task（任务）
一句话说明要完成的目标。

## Input（输入）
提供给 AI 的原材料、数据、参考资料。

## Process（过程）
明确的执行步骤或思考链。

## Output（输出）
期望的输出格式、结构、交付物。
```

## 为什么有效？

### 1. 消除歧义

很多模糊的指令源于缺少上下文。C-TIPO 强制在 `Context` 中说明背景，让 AI Agent 能做出更符合预期的判断。

### 2. 过程可控

`Process` 是关键——它不是告诉 AI "做什么"，而是告诉 AI "怎么做"。这对于需要多步推理的复杂任务尤其重要。

### 3. 输出结构化

`Output` 定义了最终交付物的格式，无论是 Markdown、JSON、表格还是纯文本，AI Agent 都能精准产出。

## 一个例子

假设你想让 AI 分析一份用户反馈：

```markdown
## Context
我们是一个电商平台，最近收到了 200 条用户反馈，
需要从中提取关键问题并排序优先级。

## Task
分析用户反馈，输出 Top 5 问题及其改进建议。

## Input
[
  "配送太慢了，等了 5 天",
  "搜索功能不好用，找不到商品",
  "客服回复很及时，态度很好",
  ...
]

## Process
1. 逐条分类（配送/搜索/客服/支付/其他）
2. 统计每类频次
3. 按频次排序，选取 Top 5
4. 对每个问题给出 1-2 条改进建议

## Output
| 排名 | 问题类别 | 频次 | 改进建议 |
|------|---------|------|---------|
| 1 | 配送 | 68 | ... |
```

## 使用场景

| 场景 | 说明 |
|------|------|
| **代码生成** | Context 说明项目架构和技术栈，Process 规定实现步骤 |
| **内容创作** | Context 给出目标读者和风格，Output 规定文章结构 |
| **数据分析** | Input 提供数据集，Process 规定分析方法和步骤 |
| **代码审查** | Context 说明项目规范和约束，Output 要求输出问题和建议 |
| **决策辅助** | Context 说明当前困境和限制，Process 要求多角度分析 |

## 总结

C-TIPO 模板的核心思想是：**好输入 = 好输出**。通过结构化地提供 Context、Task、Input、Process 和 Output，你能让 AI Agent 更准确地理解需求，产出更一致、更可控的结果。

下次使用 AI Agent 时，不妨试试 C-TIPO 模板。
